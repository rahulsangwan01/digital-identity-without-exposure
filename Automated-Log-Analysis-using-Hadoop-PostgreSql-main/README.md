# Automated-Log-Analysis-using-Hadoop-PostgreSql
In the current world, many software system produces log files since it is a necessity among software applications to monitor real-time events. Small systems can read and analyze logs manually. However, the number of logs increases with the expansion of the system. The current world generates logs every second and this makes it hard to read and analyze such logs through the use of traditional means. The Hadoop and PostgreSQL could be used as the Big Data technologies to automate the log analysis.
The point of it is to create an Automated Log Analysis System with Hadoop and PostgreSQL that is capable of processing massive log files in an organized way. The secondary goals are scalability of the system to accommodate the growing amount of data and the adoption of an architecture to generate reports.
The plan of action is simply to develop a hybrid architecture with PostgreSQL as the structured result format and fast querying and Hadoop as the large-scale processing format. These raw logs are acquired in various sources and processed by use of procedures such as MapReduce. Aggregation functions are also carried out to compute results. The end product is then passed on to PostgreSQL so that it could be accessed more readily and quicker reporting.
